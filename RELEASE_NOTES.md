## v3.0.0 (2026-05-02) — "Kernel Reborn"

> v3.0.0 是项目历史上最大规模的架构重构，历时 7 个阶段，彻底重建了底层内核架构，引入依赖注入容器、能力化架构和自愈系统。

### 架构重构

#### Phase 1: 基础层 (Foundation Layer)

- **DI Container:** 基于依赖注入的服务容器，统一管理所有核心服务的生命周期
- **EventBus v3:** 全新事件总线，支持事件溯源、Schema 验证和死信队列
- **ServiceRegistry:** 服务注册表，提供声明式的服务发现和依赖管理

#### Phase 2: MCP 路由瘦身

- 通过自动发现机制替代手动注册，大幅精简 MCP 路由代码
- 减少样板代码，提升可维护性

#### Phase 3: tRPC 路由对齐

- 将 tRPC 路由统一接入 DI 容器
- 确保所有 API 层共享一致的服务实例和依赖关系

#### Phase 4: 七大能力架构 (Capability Architecture)

引入模块化的能力体系，将系统功能划分为七大独立能力域：

| 能力域 | 职责 |
|--------|------|
| **Task** | 任务管理核心 |
| **AI** | AI 引擎与智能分析 |
| **Workflow** | 工作流引擎 |
| **Notification** | 通知推送 |
| **Integration** | 外部集成 |
| **Agent** | 智能体管理 |
| **Observability** | 可观测性 |

#### Phase 5: 自愈系统 (Self-Healing System)

- **健康检查:** 实时监控各核心服务健康状态
- **熔断器 (Circuit Breaker):** 故障自动隔离，防止级联失败
- **死信队列 (DLQ):** 处理失败事件的持久化与重试
- **SSE 推送:** 运维状态实时推送至前端

#### Phase 6-7: 运维面板 (Ops Panel)

全新运维面板，包含 **8 个视图**：

1. **总览 (Overview)** — 系统全局状态一览
2. **拓扑 (Topology)** — 服务依赖关系可视化
3. **联动 (Linkage)** — 服务间联动关系
4. **事件 (Events)** — 实时事件流
5. **工作流 (Workflows)** — 工作流执行监控
6. **AI** — AI 引擎状态与调用统计
7. **通知 (Notifications)** — 通知推送状态
8. **干预 (Interventions)** — 人工干预操作
